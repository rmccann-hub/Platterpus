# Runtime deps bundled into the AppImage. Versions track DEPENDENCIES.md.
#
# python-appimage runs `pip install` once per line below (from a temporary
# build directory), so a global `--find-links .` line does NOT work here —
# it would be its own argument-less install and pip would also resolve `.`
# against the wrong directory. Instead, build_appimage.sh exports
# PIP_FIND_LINKS pointing at this recipe directory, so the `platterpus`
# line resolves to the locally-built platterpus-X.Y.Z-*.whl rather than
# going to PyPI (PIP_FIND_LINKS is a pip env var, so it survives pip's
# `-I` isolated mode, unlike PYTHON* variables).

# NOTE on version operators: python-appimage installs each line with a
# shell (`system()` does `' '.join(args)` + `shell=True`), so a `<` or `>`
# in a specifier is read as a shell redirection (e.g. `,<7` → "open file 7"
# → build crash). We therefore use the compatible-release operator `~=`,
# which expresses the same bounds without any shell-special characters:
#   PySide6 ~=6.7  ==  >=6.7,<7.0      tomli-w ~=1.0  ==  >=1.0,<2.0
#
# LIMITATION — no `pip --require-hashes` byte-pinning here (tracked). Full
# reproducibility pins each third-party dependency to an exact version + sha256.
# The naive `--require-hashes` per line can't work here (python-appimage installs
# each line in its own `pip install`, and hash mode then demands hashes for that
# line's TRANSITIVE deps too — e.g. PySide6 → shiboken6 — unavailable in a
# single-line install). The python-appimage-compatible solution SHIPPED: a
# separate step pre-downloads the full closure into a hash-verified wheelhouse
# and the per-line installs then resolve OFFLINE from it (build_appimage.sh's
# PIP_FIND_LINKS block, activated when requirements.lock exists — regenerate it
# with build/lock-requirements.sh). The `platterpus` line stays a locally-built,
# unhashed wheel (its bytes are pinned by SOURCE_DATE_EPOCH instead). Timestamps
# are also pinned (SOURCE_DATE_EPOCH). Remaining: validate full-AppImage byte
# reproducibility on a real release build (only the wheel half is sandbox-checkable).
PySide6~=6.7
musicbrainzngs==0.7.1
tomli-w~=1.0
# Ed25519 release-signature verification (update_signing.py). `~=` can't span
# majors, so this pins the bundled line to one recent series while pyproject
# allows the broader >=41,<47 for dev/pipx; the exact bundled version is fixed
# by requirements.lock (see the reproducibility note above). Bump both together.
cryptography~=44.0
