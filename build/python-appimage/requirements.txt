# Runtime deps bundled into the AppImage. Versions track DEPENDENCIES.md.
#
# python-appimage runs `pip install` once per line below (from a temporary
# build directory), so a global `--find-links .` line does NOT work here —
# it would be its own argument-less install and pip would also resolve `.`
# against the wrong directory. Instead, build_appimage.sh exports
# PIP_FIND_LINKS pointing at this recipe directory, so the `platterpus`
# line resolves to the locally-built platterpus-X.Y.Z-*.whl rather than
# going to PyPI (PIP_FIND_LINKS is a pip env var, so it survives pip's
# `-I` isolated mode, unlike PYTHON* variables).

# NOTE on version operators: python-appimage installs each line with a
# shell (`system()` does `' '.join(args)` + `shell=True`), so a `<` or `>`
# in a specifier is read as a shell redirection (e.g. `,<7` → "open file 7"
# → build crash). We therefore use the compatible-release operator `~=`,
# which expresses the same bounds without any shell-special characters:
#   PySide6 ~=6.7  ==  >=6.7,<7.0      tomli-w ~=1.0  ==  >=1.0,<2.0
#
# LIMITATION — no `pip --require-hashes` byte-pinning here (tracked). Full
# reproducibility would pin each dependency to an exact version + sha256
# (`pip-compile --generate-hashes`). That can't be expressed in THIS recipe:
# python-appimage installs each line in its own shell `pip install`, and
# `--require-hashes` mode then demands a hash for that line's TRANSITIVE deps
# too (e.g. PySide6 → shiboken6), which aren't available in a single-line
# install — and the `platterpus` line is a locally-built, unhashed wheel that
# changes every commit. Timestamps ARE pinned (SOURCE_DATE_EPOCH in
# build_appimage.sh); exact dependency bytes are the remaining gap. Revisiting
# needs a python-appimage-compatible pre-resolved/hashed install step.
PySide6~=6.7
musicbrainzngs==0.7.1
tomli-w~=1.0
platterpus
